# Delete Your AgentKip Account

Status: **BLOCKED - intended user guide; deletion UI/route is not implemented**

Do not publish this guide until the described flow and system-by-system deletion
evidence pass. Account deletion must be available to every user if AgentKip
creates accounts.

## Intended release flow

1. Open **Settings > Legal & Privacy > Delete Account**.
2. Review what AgentKip will delete, what it must temporarily retain, what remains
   in systems you control, and any subscription effect.
3. Reauthenticate and confirm deletion.
4. Save the request/status reference if completion takes time.
5. Check the final completed or partial-completion notice.

## What the request covers

Subject to the final verified inventory, the request must cover AgentKip-controlled
account, authentication, consent, entitlement, integration-token, support,
diagnostic, and service records. Any retained category must be named with its
reason and expected deletion/expiry behavior.

## What it does not delete automatically

- GitHub repositories, organizations, issues, pull requests, or GitHub account.
- Apps, phones, computers, devices, deployments, domains, or other infrastructure.
- Accounts or records independently controlled by an AI/model provider.
- Self-hosted Noggin data unless you separately request and verify deletion there.
- Apple purchase/subscription records controlled by Apple.
- Records another party must retain under its own terms or legal obligations.

Disconnect or delete those resources with the system that controls them. AgentKip
must not imply that an external deletion succeeded without confirmation.

## If deletion is pending or partial

The status must identify the remaining system, what AgentKip is doing next, and
whether you need to act. A local sign-out or removed screen is not proof that
server/provider data was deleted.

For help regaining account access to use the in-app flow, contact
`hello@agentkip.ai`. Support must not ask for your password, recovery key, API
key, or repository secret.
